## 問題の症状

4週間前、Claude Codeを使って仮想通貨の自動売買Botを作った。

構成はこんな感じ。

- 取引所: Zaif / bitFlyer / Coincheck / bitbank / Binance Japan の5社
- Bot数: 6台（戦略別に切り替え）
- 戦略: DCA、ルールベース、スキャルピング、グリッド
- 実行間隔: 1〜5分

動き始めて2週間経ったあたりで、気づいた。

**「これ、確定申告どうするんだ？」**

損益レポートを見てみたら、**週あたり68件の取引**が発生していた。ここから年間換算すると約3,500件。スキャルピング系の戦略を追加したらさらに増える見込みで、年間5,000〜10,000件は固い。

その時点で、私のExcelには既に200行超の取引履歴が並んでいた。手動で1件ずつ計算することを考えた瞬間、背筋が凍った。

```
2026-04-01 BUY  BTC/JPY  0.0001  ¥11,136,095  手数料¥11
2026-04-01 SELL BTC/JPY  0.0001  ¥11,158,234  手数料¥11  PnL +¥11
2026-04-01 BUY  ETH/JPY  0.01    ¥552,103     手数料¥5
...（この時点で200行、最終的に3,000行以上になる予定）
```

日本の仮想通貨税制は **総平均法** or **移動平均法** での計算が必要で、取引ごとに取得単価を再計算しなきゃいけない。取引所ごとに手数料の扱いも違う。Binance Japanは0.1%、Coincheckは0%（取引所方式）、Zaifは銘柄によって変わる。

手動でこれを全部やるのは、完全に狂気の沙汰だった。

## 原因調査

調べると、仮想通貨の確定申告には以下の選択肢があることが分かった。

### 選択肢1: Excelで手動計算

年間100件以下なら何とかなるかもしれない。しかしBotで自動売買している時点で、**数千件単位の取引**が発生する。手動計算は物理的に不可能。

仮に1件あたり1分で計算できたとして、3,000件 × 1分 = 50時間。**まるまる2日潰れる。** しかもミスしたら全部やり直し。

### 選択肢2: 税理士に丸投げ

税理士に依頼すれば確実だが、仮想通貨に詳しい税理士は少なく、**費用も10〜30万円**かかる。個人のBot運用で年間10万円の税理士費用は、収支的に厳しい。

### 選択肢3: 損益計算ツールを使う

仮想通貨の損益計算を自動化するツールがいくつか存在する。取引履歴のCSVをアップロードすると、総平均法・移動平均法で自動計算してくれる。

この中で最もメジャーなのが **クリプタクト** だった。

> [税理士も使う、完全無料の仮想通貨税金計算サービス【CRYPTACT（クリプタクト）】](https://px.a8.net/svt/ejp?a8mat=4B1EPO+16V8C2+4DGW+5Z6WY)

税理士も使うという触れ込みだったので、信頼性は高そうだと判断した。

### クリプタクトを選んだ理由

他にもツールはあったが、最終的にクリプタクトに決めた理由は以下の3つ。

| 理由 | 内容 |
|------|------|
| **対応取引所の多さ** | 国内・海外100以上の取引所に対応。Botが使っている5社全てに対応していた |
| **無料プランがある** | 年間50件までは無料。Botの本格運用前に試せる |
| **API連携あり** | 手動でCSVをアップしなくても、API経由で取引履歴を同期できる |

Botで運用している以上、**API連携は必須**だった。毎週CSVをダウンロードしてアップロードする運用は続かない。

## 解決策コード

### クリプタクトの使い方（実装者視点）

やることはシンプルだった。

**Step 1: アカウント登録**

> [取引記録をアップするだけで自動で税金計算【仮想通貨損益計算のCRYPTACT（クリプタクト）】](https://px.a8.net/svt/ejp?a8mat=4B1EPO+16V8C2+4DGW+5ZEMQ)

メールアドレスで登録。無料プランで始められる。

**Step 2: 取引所のAPI連携**

各取引所のReadOnly APIキーを発行して、クリプタクトの設定画面から登録する。これで自動同期が始まる。

Bot側で発行しているAPIキーとは**別のキー**を作るのがポイント。権限を `read` だけに絞っておけば、万が一漏洩しても被害はない。

```python
# Bot側のAPIキー（trade権限あり）
BOT_API_KEY = "xxx"  # 注文発行に使う

# クリプタクト用のAPIキー（read権限のみ）
TAX_API_KEY = "yyy"  # 残高・取引履歴の閲覧のみ
```

Zaifはこの辺の権限設定が分かりにくくて30分ほど迷った。取引所ごとにUIが全然違うので、API連携するツールを使うなら最初にこの作業だけは人力で頑張る必要がある。

**Step 3: 計算方法の選択**

総平均法か移動平均法を選ぶ。**一度選ぶと変更不可**なので注意。

一般的な個人トレーダーなら**総平均法**が楽。年末に1回計算するだけで済む。移動平均法は取引の度に平均単価を再計算する必要があり、手動では現実的じゃない。

### Bot運用との相性

クリプタクトとBotを併用する上で、気をつけたポイント。

#### 同期頻度

クリプタクトは取引履歴を**毎日1回**同期する。Bot側でリアルタイムの損益を確認したい場合は、Bot自身のDBを見るしかない（クリプタクトは確定申告向けのサービスなので、リアルタイム性は重視していない）。

```python
# Bot側のリアルタイム損益（SELLしたtradeから集計）
total_pnl = db.query(func.sum(Trade.pnl)).filter(
    Trade.bot_id == bot.id,
    Trade.trade_type == "SELL"
).scalar() or 0
```

#### 手数料の扱い

クリプタクトは取引所ごとの手数料を自動で計算してくれる。Bot側でも `手数料込みのPnL` を計算する必要があるが、**最終的な確定申告用の数字はクリプタクト側の計算が正**と割り切っている。

自前計算と微妙にズレることがあるが、これは取引所の手数料レートが変動するため。Bot側は参考値、確定申告はクリプタクトで、という役割分担にしている。

#### マルチアカウント対応

Botを複数取引所で動かしているので、**全取引所の損益を一元管理**できるのは非常に楽。クリプタクトは5社分の取引履歴を自動でマージして、トータルの損益を計算してくれる。

これを手動でExcelにまとめようとすると、取引所ごとに書式が違うCSVを整形するだけで丸1日かかる。

### 結果

年間取引件数が爆増する前提の運用でも、確定申告の準備時間はほぼゼロになった。

| 項目 | Before（手動） | After（クリプタクト） |
|------|-------------|-------------------|
| 取引履歴の集約 | 丸1日 | 自動同期（0分） |
| 損益計算 | 50時間 | 1クリック（数秒） |
| 税額計算 | 要税理士相談 | 自動表示 |
| 年末の作業時間 | 3日 | 30分 |

確定申告のためにBotを止める必要はなくなった。**これはBot運用者にとって革命だった。**
