自分は C 言語を 23 年間勉強してきました。フリーランスで組み込みシステムのコードレビューを行うことが多く、安全で信頼性の高いコードを書くことが大切なことです。しかし、コードの複雑さや時間の制約により、人間のレビュワーがすべての危険性を発見することは難しいことがあります。そこで、静的解析ツールである creview を使って危険コードを検出する方法について紹介します。

## この記事の前提（SCQA）

- **状況**: C 言語のコードレビューで、安全性と信頼性を確保することが重要です。人間のレビュワーだけではすべての危険性を発見することが難しいことがあります。
- **問題**: コードの複雑さや時間の制約により、人間のレビュワーがすべての危険性を発見することが難しいことがあります。
- **問い**: どのようにしてコードの安全性と信頼性を確保することができるか？
- **答え**: 静的解析ツールである creview を使って危険コードを検出することができます。

## 検証用コード

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

/* スタックバッファのオーバーフロー（strcpy + sizeof 誤用） */
void copy_name(const char *src) {
    char buf[16];
    strcpy(buf, src);                    /* 検出すべき: 長さ未検証 */
    printf("name=%s\n", buf);
}

/* sprintf による境界未検証 */
void format_path(const char *dir, const char *file) {
    char path[32];
    sprintf(path, "%s/%s", dir, file);   /* 検出すべき: snprintf を使うべき */
}

/* sizeof(ptr) ハマリパターン */
void wipe_buffer(char *p) {
    memset(p, 0, sizeof(p));             /* 検出すべき: sizeof(p) は常に 8（ポインタサイズ） */
}

/* 配列インデックス未検証（scanf 入力をそのまま添字に） */
void read_at(int *arr) {
    int i;
    scanf("%d", &i);
    printf("%d\n", arr[i]);              /* 検出すべき: i の上下限チェック無し */
}
```

## creview の使い方（CLI 実行例）

```
L7 【保守危険】: マジックナンバー16。定数定義なしで意味不明・変更困難
L8 【重大】: strcpy使用。コピー元長未検証でオーバーフロー可能
L14 【保守危険】: マジックナンバー32。定数定義なしで意味不明・変更困難
L15 【重大】: sprintf使用。出力バッファ長未検証でオーバーフロー可能
L20 【重大】: pのNULL未検証でmemset実行。NULLポインタ渡しでクラッシュ
L27 【重大】: 外部入力iを境界チェックなしで配列添字に使用。範囲外アクセス
```

上記のラベル（【重大】【設計不明】【保守危険】）は、コードの安全性と信頼性に影響を与えるレベルを示しています。【重大】は最も深刻な問題であり、すぐに修正する必要があります。【設計不明】はコードの設計に問題があることを示し、【保守危険】はコードの保守性に問題があることを示します。

## まとめ

* creview は C 言語のコードを静的解析するツールです。
* さまざまな CWE 番号に対応した危険コードを検出できます。
* コードの安全性と信頼性を確保するために、creview を使うことができます。
* c-review-ai、creview、CSAF の 3 つのツールを組み合わせて、コードの安全性と信頼性を確保することができます。
